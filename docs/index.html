How it works

Three signals. One score. One decision.

Every pull request gets analysed across three dimensions of code quality, then scored 0‚Äì100. Below 70? Blocked.

üåÄ

Cyclomatic Complexity

Runs lizard (Python) and escomplex (JS/TS) to measure function complexity. AI models write code that works but doesn't refactor ‚Äî this catches it.

üß™

Test Coverage

Branch + line coverage via coverage.py. AI-generated tests often test happy paths only. Branch coverage exposes the missing edge cases.

üîç

AI Anti-Pattern Scan

Custom Semgrep rules for patterns AI models produce disproportionately: bare except, SQL via f-strings, hardcoded secrets, mutable defaults, eval(), and more.

üêõ

Fuzz Testing

Register fuzz harnesses against source globs in fuzz/targets.txt. When a PR touches those files, atheris runs them automatically ‚Äî crashes block the merge.

üìä

Score History

Every run is stored in a lightweight SQLite database. Track score trends per repo and branch over time ‚Äî see if your codebase is getting better or worse.

üîî

Slack Notifications

Send pass/fail notifications to a Slack channel or any webhook endpoint. Keep the whole team informed when AI debt is accumulating.

Installation

Running in 2 minutes

No SaaS account. No data leaves your repo. One workflow file.

1

Copy the workflow

Add the GitHub Actions workflow to your repository. It runs automatically on every pull request.

2

Set your threshold

The default is 70/100. Tune it in pyproject.toml or via the threshold input ‚Äî no code changes needed.

3

Add pre-commit hooks

Optional but recommended: block high-complexity functions and AI anti-patterns at commit time, before they ever reach CI.

# .github/workflows/ai-gate.yml
name: AI Code Quality Gate
on:
  pull_request:
    branches: [main, master, develop]

permissions:
  contents: read
  pull-requests: write   # needed to post PR comment

jobs:
  gate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with: { fetch-depth: 0 }

      - name: Run AI Code Gate
        uses: your-org/ai-code-gate@v1
        with:
          threshold: "70"
          languages: "python"   # or "python,javascript,typescript"
          post-comment: "true"
          # slack-webhook-url: ${{ secrets.SLACK_WEBHOOK_URL }}

Want to tune thresholds without editing YAML? Add a [tool.ai-gate] section to your pyproject.toml: 

# pyproject.toml
[tool.ai-gate]
threshold = 75

[tool.ai-gate.complexity]
excellent   = 5
acceptable  = 10
poor        = 20

[tool.ai-gate.coverage]
excellent   = 90
acceptable  = 70
poor        = 40

[tool.ai-gate.antipatterns]
excellent   = 0
acceptable  = 2
poor        = 5
